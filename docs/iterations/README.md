## 🔄 迭代原则

### 每个迭代的标准流程

1. **计划阶段** (10%)
   - 明确迭代目标
   - 列出具体任务
   - 确定验收标准

2. **开发阶段** (70%)
   - 按照任务清单开发
   - 每次只做一个功能点
   - 及时提交代码

3. **测试阶段** (15%)
   - 手动测试验证
   - 修复发现的问题

4. **总结阶段** (5%)
   - 更新文档
   - 记录经验教训
   - 准备下一个迭代

### 迭代完成标准

- ✅ 所有计划功能已实现
- ✅ 功能通过测试
- ✅ 文档已更新
- ✅ 可以独立演示
